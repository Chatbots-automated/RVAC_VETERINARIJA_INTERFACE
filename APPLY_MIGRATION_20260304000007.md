# Apply Migration for Comments and Non-Driving Hours

## Important: Database Migration Required - UPDATED VERSION!

The hours calculation for `vairuotojas` will not work correctly until you apply this migration!

**If you already applied the migration once, you MUST apply it again!** The migration has been updated to properly handle the `hours_worked` column.

Currently showing: **3h** (calculated from start/end times: 08:10-12:10 = 4h - 1h lunch = 3h)
After migration: **5h** (from "Valandos nevairuojant" field)

## Steps to Apply Migration

1. Go to your Supabase Dashboard: https://supabase.com/dashboard
2. Navigate to your project
3. Go to **SQL Editor** in the left sidebar
4. Click **New Query**
5. Copy and paste the entire contents of:
   ```
   supabase/migrations/20260304000007_add_comments_and_non_driving_hours.sql
   ```
6. Click **Run** or press `Ctrl+Enter`
7. Verify success message appears
8. **IMPORTANT:** After applying, you may need to re-enter the data for vairuotojas workers OR manually update existing rows

## What This Migration Does

1. **Adds `comments` column** - Allows users to add notes/comments for each work day
2. **Adds `non_driving_hours` column** - For vairuotojas, stores the actual hours worked (not driving time)
3. **Updates `hours_worked` calculation** - For vairuotojas, uses `non_driving_hours` instead of calculating from start/end times

## After Migration

- For **vairuotojas**: Start/End times are still entered (for record keeping) but hours are taken from "Valandos nevairuojant" field
- For **darbuotojas** and **traktorininkas**: Hours are still calculated from start/end times with lunch deduction
- Comments field will be available for all worker types

## Verification

After applying the migration, refresh your application and:
1. Check that the hours for vairuotojas show correctly (should match "Valandos nevairuojant" value)
2. Verify comments field is working
3. Test that the total hours in Perziura section are correct
